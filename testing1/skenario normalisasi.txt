ðŸ”¹ Langkah 1: Hilangkan Bagian Tidak Penting

Hilangkan informasi yang tidak relevan atau terlalu spesifik, seperti:

Timestamps (jika tidak digunakan sebagai fitur)

IP address, user ID (kecuali kamu ingin analisis user/IP)

Tujuan: agar model fokus pada pola pesan log, bukan variasi spesifik.

Contoh:

WARNING - Disk usage high

ðŸ”¹ Langkah 2: Lowercase (Huruf Kecil Semua)

Untuk konsistensi teks.

Error dan error dianggap sama.

Hasil:

info - user logged in
warning - disk usage high
error - database timeout

ðŸ”¹ Langkah 3: Tokenisasi

Pisahkan teks jadi kata-kata (token).

Misalnya: ["error", "database", "timeout"]

ðŸ”¹ Langkah 4: Stopword Removal (Opsional)

Hapus kata umum yang tidak menambah arti khusus, seperti: the, in, from, on, and, dll.

Bisa dilakukan jika kamu ingin fokus pada kata-kata penting.

ðŸ”¹ Langkah 5: Lemmatization / Stemming (Opsional)

Ubah kata ke bentuk dasarnya:

logged, logging â†’ log

timeout, timed out â†’ timeout

ðŸ”¹ Langkah 6: Representasi Angka

Setelah teks bersih, kamu ubah jadi angka agar bisa digunakan dalam model:

Beberapa metode umum:
Metode	Penjelasan Sederhana
TF-IDF	Hitung seberapa penting kata dalam log
Bag of Words	Hitung berapa kali kata muncul
Word Embedding (Word2Vec, GloVe)	Ubah kata jadi vektor yang punya makna
BERT Embedding	Lebih akurat, tapi kompleks (pakai model pretrained)
ðŸ”¹ Langkah 7: Normalisasi Nilai Numerik (Jika Ada)

Kalau log kamu punya angka (misalnya: CPU usage, waktu eksekusi, ukuran file), normalisasi angkanya:

Gunakan:

Min-Max Scaling â†’ ubah ke rentang 0-1

Z-Score / StandardScaler â†’ ubah agar punya mean = 0 dan std = 1
